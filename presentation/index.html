Technical · the transform engine

Agents as composable config

Skip imperative — the interesting part is generative: run an agent with source + the change as context. Composed in Starlark, familiar to Bazel / Buck2 shops.
harness(name="claude_code", kind="claude_code", plugins=["//tools/plugins:bazel"])
model(name="opus", provider="anthropic", id="claude-opus-4-8")
agent(name="porter", harness="//tools/harness:claude_code", model="//tools/models:opus")

agent_transform(agent="//tools/agent:porter",
                prompt=template("//tools/agent/prompts:port"))
harness × model × prompt output → content-addressed patch reproducible
Side effect: config-as-code + reproducible output lets you 📊 replay history to benchmark agents/models (cost·speed·quality) and 🐛 git bisect regressions.
Use cases

Transform code in & out

Import · outer world → monorepo

  • 🔍 Scan — security / license over every change
  • ⚙️ Optimize to a goal — “tune these kernel changes for my datacenter's hardware profiles”
  • 🔁 Migrate — rewrite our app vs upstream's new API

Export · monorepo → outer world

  • 📤 “update the OSS demo app + docs to match my new SDK” — before the PR goes out

Same vocabulary, both directions.

Agents propose, Bazel CI disposes — transformed changes are verified imperatively.
